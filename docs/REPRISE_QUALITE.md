# Plan de reprise qualite

## Constat

La base GitHub est stable pour Render, mais elle doit retrouver la qualite des versions precedentes.

Points a reprendre :

- identite visuelle de Rituels d'Ailleurs ;
- planning professionnel ;
- ILU general avance ;
- reservation plus fluide ;
- gestion des cabines ;
- routines type Fabriq ;
- espace client.

## Regle de marque

Le site public affiche l'entreprise : Rituels d'Ailleurs et Le Petit Rituel.

Le nom technique du logiciel ne doit pas remplacer le nom de l'entreprise cote public.

## Priorites

### 1. Design public

- accueil premium ;
- navigation claire ;
- cartes prestations ;
- responsive mobile ;
- charte beige, doree, blanc casse et brun naturel.

### 2. Planning

- vue ressources ;
- lignes par praticienne et cabine ;
- axe horaire ;
- blocs rendez-vous, presence, absence, conge, formation ;
- planning date ;
- repos variables ;
- samedis tournants.

### 3. ILU

- tableau salaries x prestations ;
- vide = ne connait pas ;
- I = non reservable ;
- L = reservable ;
- U = reservable ;
- filtres par institut et categorie ;
- couleurs par niveau.

### 4. Reservation

Parcours cible : prestation, calendrier mensuel, jour disponible, horaire, confirmation.

Le moteur controle : planning reel, ILU minimum L, cabine compatible, absence de chevauchement et duree suffisante.

### 5. Cabines

- compatibilite prestation/cabine ;
- types de cabine ;
- indisponibilites ;
- QR code ;
- routines associees.

### 6. Routines

- checklists ;
- photos ;
- commentaires ;
- rapports ;
- historique par utilisateur connecte.

## Ordre de developpement

1. identite et design ;
2. planning professionnel ;
3. ILU avance ;
4. reservation avec cabines compatibles ;
5. routines ;
6. espace client ;
7. stocks avances ;
8. dashboard direction.
